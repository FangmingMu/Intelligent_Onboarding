# Stage 2: 工业级 Hybrid RAG 检索引擎

本模块负责从海量私有文档中精准提取知识。通过架构演进，系统已具备处理企业级复杂术语与硬核数据的能力。

---

## 🛠️ 核心架构特性

### 1. 混合检索策略 (Hybrid Search)
*   **BM25 (关键词)**：擅长处理 IP 地址、版本号、VPN 域名等强特征。
*   **Dense Vector (向量)**：擅长处理语义相近但措辞不同的自然语言问询。
*   **Ensemble**：通过加权融合，在 k=7 实验中，混合检索的 **Context Precision 提升了 5%**，有效解决了单一向量检索在专有名词上的“语义模糊”问题。

### 2. 两阶段检索链路
*   **召回 (Recall)**：极速从 FAISS 库中提取 Top-10 候选片段。
*   **精排 (Rerank)**：利用 **BGE-Reranker** 对候选片段进行二次语义打分。
*   **价值**：确保注入 Prompt 的上下文具备极高的相关性，最大程度抑制模型幻觉。

### 3. Ragas 自动化评测体系
系统内置了基于 **Ragas** 的量化评估体系，涵盖四个核心维度：
*   **Faithfulness (忠实度)**：评估答案是否忠于原文。
*   **Answer Relevancy (相关性)**：评估答案是否切中用户需求。
*   **Context Recall (召回率)**：基于 20 组黄金数据集评估知识覆盖面。
*   **Context Precision (精度)**：评估重排后的排序质量。

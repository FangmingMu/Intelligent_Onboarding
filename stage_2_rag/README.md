# Stage 2: 语义搜索与 RAG 引擎 (RAG Engine)

## 核心职责
基于企业私有文档（Markdown）提供精准、可溯源的知识问答服务。

## 技术栈
- **MarkdownHeaderTextSplitter**: 实现按标题层级的语义切分。
- **FAISS**: 高性能本地向量索引库。
- **BGE-Reranker**: 深度学习重排模型，提升检索精度。
- **Qwen3-Embedding**: 针对中文优化的向量嵌入模型。

## 关键流程
1. **语义切分**: 保持段落逻辑完整性，注入层级元数据。
2. **两阶段检索**: 
   - **召回 (Recall)**: 基于向量相似度快速锁定候选片段。
   - **重排 (Rerank)**: 利用精排模型解决“搜得到但排不准”的问题。
3. **知识注入**: 将检索结果作为上下文注入 Prompt，强制模型基于事实回答。

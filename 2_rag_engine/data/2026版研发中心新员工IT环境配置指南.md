# [机密] 2026版研发中心新员工IT环境配置指南 (V4.1)

## 0. 概述
本指南旨在规范化研发中心（R&D Center）新入职员工的基础设施接入流程。所有配置动作必须严格遵循本手册，严禁私自绕过 `Security-Gateway-V3` 进行非合规操作。

## 1. 基础网络接入与加密链路 (VPN-Sec-Tunnel)
在进行任何系统访问前，必须建立基于双因子验证（2FA）的内网加密隧道。

1. **内网VPN-Sec客户端安装**
    1.1 从 `Internal-Nexus-Repo` 下载对应的 `AnyConnect-Custom-V2026` 客户端。
    1.2 导入 `Root-CA-Cert-2026.crt` 根证书，确保证书链合法性。
    1.3 在配置界面输入网关地址：`vpn-gw-shanghai.corp-sec.com:4433`。
2. **多因子身份认证 (MFA-Auth)**
    2.1 打开 `Corp-Authenticator` App，扫描网页端生成的 `Secret-QR-Code`。
    2.2 在登录框输入 `LDAP-Account` 与动态产生的 6 位 `TOTP-Token`。
    2.3 若提示 `Auth-Protocol-Error`，需检查本地系统时钟是否与 `NTP-Server-Cluster` 同步。

## 2. 研发基础设施与权限授权 (Jira-Auth-Token)
所有研发活动必须在受监控的环境下进行。

1. **Jira 与 Confluence 接入**
    1.1 访问 `Jira-Center.corp.internal`。
    1.2 在个人设置中生成一个长期有效的 `Jira-Auth-Token-V2`。
    1.3 将此 Token 注入到本地的 `GitLab-Runner-Config` 环境变量中，以实现 Webhook 自动触发。
2. **代码仓库与 CI/CD 管道 (GitLab-Runner-V4)**
    2.1 全局配置 Git 用户信息：
        - `git config --global user.email "Your-LDAP-ID@corp.internal"`
    2.2 生成 SSH 密钥对（必须使用 `ed25519` 算法）：
        2.2.1 运行 `ssh-keygen -t ed25519 -C "Dev-Terminal-Cert"`。
        2.2.2 将公钥内容粘贴至 GitLab 个人 Profile 的 `SSH-Keys-Vault`。
    2.3 测试连接：`ssh -T git@gitlab-cluster.internal`，确认返回 `Welcome-Auth-Success`。

## 3. 云原生开发环境 (K8s-ArgoCD-Sync)
研发中心所有微服务均部署于私有云 K8s 集群。

1. **集群访问令牌获取**
    1.1 使用 `Kubectl-Login-Helper` 登录集群网关 `K8s-Ingress-GW`。
    1.2 选择所属命名空间（Namespace），通常为 `dev-group-{LDAP-ID}`。
2. **自动化部署流程 (GitOps)**
    2.1 访问 `ArgoCD-Console.internal` 查看应用状态。
    2.2 触发 `K8s-ArgoCD-Sync` 手动同步动作：
        2.2.1 确认 `Target-Revision` 为当前最新的 `Commit-ID`。
        2.2.2 点击 `Sync` 并勾选 `Prune-Resources`。
    2.3 监控 Pod 状态：`kubectl get pods -l app=core-service --watch`。

## 4. 故障排查与专用名词索引 (Glossary)
- **IDC-LB-Vip**: 互联网数据中心负载均衡虚拟IP地址。
- **Nexus-Private-Repo**: 内部私有制品库，用于存储 Docker 镜像及 Maven/NPM 包。
- **Prometheus-AlertManager-Relay**: 监控指标告警转发中继器。
- **WAF-Protection-Rule**: Web 应用防火墙动态过滤规则。

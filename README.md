# 💼 企业级“文档驱动”智能 Onboarding 与工单拦截系统

一个针对企业内部新员工或 IT 支持的智能 Agent，旨在通过 RAG（检索增强生成）解决高频咨询，并利用 Tool Calling（工具调用）自动化处理基础工单。

## 1. 项目愿景与痛点解决

在大型企业中，新员工入职（Onboarding）通常伴随着海量的重复性问题，而 IT/HR 部门则被大量低价值的“修改密码”、“申请权限”等工单困扰。

本项目旨在：

* **拦截工单**：通过智能问答自动解决 60% 以上的基础 IT 咨询，减少人工干预。
* **知识沉淀**：将散落在不同部门、不同格式（PDF, Wiki, Markdown）的文档转化为可即时检索的语义知识库。
* **流程自动化**：将对话直接转化为系统动作（如 Jira 提单、OA 审批触发），实现“所言即所得”。

## 2. 核心架构设计

系统采用四层解耦架构，确保生产级的稳定性与可扩展性：

1. **接入与路由层 (Gateway & Routing)**

   * **意图识别**：对用户输入进行轻量级路由，区分“知识问答”、“业务办理”与“通用闲聊”。
   * **安全防御**：初步过滤敏感指令与非法输入。
2. **高级 RAG 引擎 (Advanced RAG Engine)**

   * **文档解析**：深度处理带表格、层级结构的复杂企业文档。
   * **语义切分 (Semantic Chunking)**：按语义段落切分，保留上下文连贯性。
   * **混合检索 (Hybrid Search)**：结合向量搜索（语义）与 BM25（关键词/专有名词）。
   * **重排机制 (Rerank)**：对召回结果进行二次精排，提升 Top-K 准确度。
3. **动作执行与安全层 (Action & Harness)**

   * **Tool Calling**：动态调用外部 API（Jira, 企业微信, LDAP 等）。
   * **结构化输出**：强制模型输出符合 Pydantic 定义的 JSON Schema。
   * **Human-in-the-loop**：针对高危权限申请，生成审批卡片并挂起任务，等待人工确认。
4. **可观测性与评测层 (Observability & Eval)**

   * **全链路追踪**：记录检索耗时、Token 消耗及路由决策路径（接入 Langfuse）。
   * **反馈闭环**：基于用户点赞/踩的数据，持续优化检索策略与知识库。

## 3. 技术栈选型

*   **核心语言**：Python 3.9+
* **Agent 框架**：LlamaIndex (RAG 调度) + 原生 SDK (状态机控制)
* **向量存储**：Qdrant / Milvus
* **算法模型**：BGE-m3 (Embedding) + BGE-Reranker-v2 (Rerank)
* **工程工具**：Pydantic (数据校验), Tenacity (防御性重试), FastAPI (接口服务)
* **监控平台**：Langfuse
